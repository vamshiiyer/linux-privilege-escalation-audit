# Linux Privilege Escalation Audit

## Overview
This project focuses on auditing Linux systems for common misconfigurations
that could potentially lead to privilege escalation. It is designed purely
for defensive security research and system hardening awareness.

## Objective
- Identify risky system configurations
- Understand common privilege escalation vectors
- Improve Linux system security posture

## Scope
- Enumeration and analysis only
- No exploitation performed
- No unauthorized access

## Areas Covered
- SUID/SGID binaries
- File permissions
- Kernel and service configurations
- Environment variables

## Ethical Notice
This project is intended strictly for educational and defensive purposes.
It must only be used on systems you own or have explicit permission to audit.
